01

議会議論データ

議会から発言までの階層構造で、「誰が、いつ、どの会議で、何を発言したか」を記録します。

{{- /* 階層図解 */ -}}
議会（Conference）
例：第213回国会・衆議院
名称 所属議会体 期
会議体（GoverningBody）
例：予算委員会、本会議
名称 種別 自治体コード
会議（Meeting）
例：2024年3月15日 予算委員会 第5号
開催日 会議名 URL
発言（Conversation）
例：「本予算案について質問いたします…」
発言者 発言内容 発言順序

この階層構造により、「ある政策テーマについて、どの会議で、誰が、どんな議論をしたか」を体系的に辿ることができます。

02

政治家データ

政治家と発言を結びつけ、所属政党や会派の履歴も含めて、政治家の活動を横断的に把握できます。

{{- /* 政治家データ関連図 */ -}}
{{- /* 中心：政治家 */ -}}
中心エンティティ

政治家（Politician）

氏名 山田 太郎
都道府県 東京都
選挙区 第1区
{{- /* 関連エンティティ群 */ -}}
発言を紐付け

発言者（Speaker）

会議での発言者情報。AI照合により政治家と自動紐付け。

氏名 所属政党名 役職 照合信頼度
所属政党

政党（PoliticalParty）

政治家が所属する政党。時期による変遷も記録。

政党名
例 自由民主党、立憲民主党、日本維新の会
会派所属

会派（ParliamentaryGroup）

議会内の投票行動を共にするグループ。政党とは異なる場合も。

会派名 所属期間 役割
例 自由民主党・無所属の会
選挙結果

選挙（Election / ElectionMember）

選挙の結果情報。得票数や当選順位を記録。

選挙日 選挙種別 得票数 当選順位

03

議案賛否データ

議案の提出から採決まで、誰が賛成し誰が反対したかを、個人単位・会派単位で記録します。

{{- /* 議案データ構造図 */ -}}
{{- /* 議案（中心） */ -}}
中心エンティティ

議案（Proposal）

議案名 令和6年度一般会計予算
種別 予算
審議状況 可決
提出日・採決日 2024-01-26 / 2024-03-28
{{- /* 提出者と採決の2カラム */ -}}
提出

提出者（ProposalSubmitter）

議案を提出した議員・会派・委員会。代表者フラグで代表提出者を区別。

提出者種別 代表者フラグ
例 内閣総理大臣（代表提出者）
採決

個人賛否（ProposalJudge）

議員個人の賛否記録。造反（党の方針と異なる投票）も検出。

賛成 反対 棄権 欠席

会派賛否（ProposalParliamentaryGroupJudge）

会派としての賛否態度。所属議員の投票と紐付け。

会派名 賛否 所属議員一覧
{{- /* Section 05 — データのつながり */ -}}
04

データのつながり

すべてのデータは、つながっています。3つのデータ群を横断して、政治活動の全体像を把握できます。

{{- /* 3データ群接続図 */ -}}
01

議会議論

議会 → 会議体 → 会議 → 発言

発言者 = 政治家
02

政治家

政治家 ← 政党 / 会派 / 選挙

議員 → 賛否投票
03

議案賛否

議案 ← 提出者 / 個人賛否 / 会派賛否

{{- /* 具体例 */ -}}

具体例：ある予算案の審議を追う

1
議案を探す

「令和6年度一般会計予算」で議案を検索

2
賛否を確認

誰が賛成し、誰が反対したか。造反した議員はいるか

3
議論を読む

反対した議員が予算委員会でどんな質疑をしたかを確認

4
政治家を深掘り

その議員の所属会派、過去の投票傾向、他の発言を横断的に調べる
