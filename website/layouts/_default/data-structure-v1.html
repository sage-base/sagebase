案1: Mermaid.js

Mermaid.jsで描く データ構造図

Mermaid.jsのER図・フローチャートを使い、エンティティ間の関連を宣言的に表現します。
コードベースで管理可能な図解アプローチ。

{{- /* Section 01 — 議会議論データ ER図 */ -}}
01

議会議論データ — 階層構造

議会から発言まで、4階層の入れ子構造をER図で表現します。 

erDiagram
    CONFERENCE["議会 (Conference)"] {
        int id PK
        string name "名称"
        string session "期"
    }
    GOVERNING_BODY["会議体 (GoverningBody)"] {
        int id PK
        string name "名称"
        string category "種別"
        string municipality_code "自治体コード"
    }
    MEETING["会議 (Meeting)"] {
        int id PK
        date meeting_date "開催日"
        string name "会議名"
        string url "URL"
    }
    MINUTES["議事録 (Minutes)"] {
        int id PK
        string content "本文"
    }
    CONVERSATION["発言 (Conversation)"] {
        int id PK
        string content "発言内容"
        int order_number "発言順序"
    }
    SPEAKER["発言者 (Speaker)"] {
        int id PK
        string name "氏名"
        string party_name "所属政党名"
        float matching_confidence "照合信頼度"
    }

    CONFERENCE ||--o{ GOVERNING_BODY : "会議体を持つ"
    GOVERNING_BODY ||--o{ MEETING : "会議を開催"
    MEETING ||--o| MINUTES : "議事録"
    MINUTES ||--o{ CONVERSATION : "発言を含む"
    CONVERSATION }o--|| SPEAKER : "発言者"

02

政治家データ — リレーション図

政治家を中心に、発言者・政党・会派・選挙の関連をER図で表現します。 

erDiagram
    POLITICIAN["政治家 (Politician)"] {
        int id PK
        string name "氏名"
        string prefecture "都道府県"
        string district "選挙区"
    }
    SPEAKER["発言者 (Speaker)"] {
        int id PK
        string name "氏名"
        float matching_confidence "照合信頼度"
    }
    POLITICAL_PARTY["政党 (PoliticalParty)"] {
        int id PK
        string name "政党名"
    }
    PARLIAMENTARY_GROUP["会派 (ParliamentaryGroup)"] {
        int id PK
        string name "会派名"
        date start_date "開始日"
        date end_date "終了日"
    }
    PG_MEMBERSHIP["会派所属 (Membership)"] {
        int id PK
        string role "役割"
        date start_date "開始日"
        date end_date "終了日"
    }
    ELECTION["選挙 (Election)"] {
        int id PK
        date election_date "選挙日"
        string election_type "種別"
    }
    ELECTION_MEMBER["選挙結果 (ElectionMember)"] {
        int id PK
        int votes "得票数"
        int rank "当選順位"
    }
    CONFERENCE_MEMBER["議会構成員 (ConferenceMember)"] {
        int id PK
        string role "役職"
    }

    POLITICIAN ||--o{ SPEAKER : "AI照合"
    POLITICIAN ||--o{ PG_MEMBERSHIP : "会派所属"
    PG_MEMBERSHIP }o--|| PARLIAMENTARY_GROUP : "所属先"
    PARLIAMENTARY_GROUP }o--o{ POLITICAL_PARTY : "構成政党"
    POLITICIAN ||--o{ ELECTION_MEMBER : "出馬"
    ELECTION ||--o{ ELECTION_MEMBER : "候補者"
    POLITICIAN ||--o{ CONFERENCE_MEMBER : "議会参加"
{{- /* Section 03 — 議案賛否データ フロー図 */ -}}
03

議案賛否データ — フロー図

議案の提出から採決までのフローと、個人賛否・会派賛否の構造を表現します。 

flowchart TD
    P["議案 (Proposal)\n議案名 / 種別 / 審議状況"]
    PS["提出者 (ProposalSubmitter)\n提出者種別 / 代表者フラグ"]
    PD["審議 (ProposalDeliberation)\n審議記録"]
    PJ["個人賛否 (ProposalJudge)\n賛成 / 反対 / 棄権 / 欠席\n造反フラグ"]
    PPG["会派賛否\n(ProposalParliamentaryGroupJudge)\n会派としての態度"]
    POL["政治家 (Politician)"]
    PG["会派 (ParliamentaryGroup)"]

    PS -->|"提出"| P
    P -->|"審議"| PD
    P -->|"採決"| PJ
    P -->|"会派態度"| PPG
    PJ -->|"投票者"| POL
    PPG -->|"所属会派"| PG
    PJ -.->|"造反検出"| PPG

    style PD fill:#fafafa,stroke:#d4d4d4,color:#171717
    style PJ fill:#fafafa,stroke:#d4d4d4,color:#171717
    style PPG fill:#fafafa,stroke:#d4d4d4,color:#171717
    style POL fill:#e5e5e5,stroke:#737373,color:#171717
    style PG fill:#e5e5e5,stroke:#737373,color:#171717
{{- /* Section 04 — 全体接続図 */ -}}
04

データのつながり — 全体俯瞰

3つのデータ群がどのように接続しているかを俯瞰します。 

flowchart LR
    subgraph DISCUSSION["議会議論データ"]
        C["議会"] --> GB["会議体"] --> M["会議"] --> CV["発言"]
    end

    subgraph POLITICIAN_DATA["政治家データ"]
        POL["政治家"]
        PP["政党"]
        PG["会派"]
        EL["選挙"]
    end

    subgraph PROPOSAL_DATA["議案賛否データ"]
        PR["議案"]
        PJ["個人賛否"]
        PPJ["会派賛否"]
    end

    CV -->|"発言者=政治家"| POL
    POL -->|"所属"| PP
    POL -->|"会派"| PG
    POL -->|"出馬"| EL
    POL -->|"投票"| PJ
    PJ -->|"対象"| PR
    PG -->|"態度"| PPJ
    PPJ -->|"対象"| PR
